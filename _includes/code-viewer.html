{%- comment -%} Full-screen code viewer. Generic — usable from any post. Call it with: include code-viewer.html file="..." label="..." href="..." Params: file (required) path under _includes/ to a markdown file holding ONE fenced code block. The fence's language tag selects the Rouge LEXER, so it must name a language Rouge actually ships. label card title. Default: "View the code". lang language NAME shown on the card. Defaults to the fence tag. Set this when the fence tag is only a stand-in lexer — Rouge has no Solidity lexer, so Solidity files fence as `javascript` and pass lang="Solidity". meta overrides the whole " · lines" line. href optional external link (gist, repo) shown in the overlay header. href_label anchor text for href. Default: "Open on GitHub". The code file is run through Liquid like any other include, so wrap its fenced block in a raw/endraw pair if the code itself contains Liquid delimiters. Note: the site is built by GitHub Pages' legacy Jekyll build, which ignores _plugins/. A custom Rouge lexer would work locally and silently no-op in production, which is why the fence tag has to be a stock Rouge language. {%- endcomment -%} {%- capture code_md -%}{%- include {{ include.file }} -%}{%- endcapture -%} {%- assign code_lines = code_md | strip | newline_to_br | split: "
" -%} {%- assign fence_tag = code_lines | first | remove: "`" | remove: "~" | strip -%} {%- assign lang = include.lang | default: fence_tag | capitalize -%} {%- assign line_count = code_lines | size | minus: 2 -%} {%- assign label = include.label | default: "View the code" -%}
